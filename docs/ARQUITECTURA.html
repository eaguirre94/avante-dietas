Módulo de Dietas Hospitalarias — AVANTE

Sistema que opera el ciclo completo de la dieta de un paciente: indicación de enfermería → oferta y confirmación del auxiliar → preparación en cocina (KDS) → entrega y cargo. Migrado al stack empresarial estándar de TI. Este documento es la guía para que cualquier persona analice el sistema sin bucear el código.

3Flujo del paciente (máquina de estados)

Cada comida recorre estos estados. Cualquier estado no terminal puede ir a CANCELADA.

ASIGNADA→ EN_OFERTA→ OFERTA_CONFIRMADA→ EN_PREPARACION→ LISTA→ EN_TRANSITO→ ENTREGADA
  • Enfermería indica la dieta (crea el paciente si no existe) → ASIGNADA.
  • Auxiliar ofrece y confirma con el paciente → OFERTA_CONFIRMADA.
  • Cocina prepara en el tablero KDS → EN_PREPARACION → LISTA.
  • Auxiliar traslada y entrega → ENTREGADA (se contabiliza el cargo).

4Módulos y roles

8 roles. Admin, chef corporativo y nutricionista son transversales (ven todas las sedes); el resto queda acotado a su sede.

Enfermería

Indicación de dieta por habitación. Regla anti-duplicado (D3).

enfermera · admin

Auxiliar

Ofrecer, confirmar, trasladar y entregar bandejas.

auxiliar · admin

Cocina (KDS)

Cola de bandejas + avisos de alergia "⚠ NO PREPARAR".

cocinera · admin

Nutrición

Catálogo de ingredientes con marca de alérgenos.

nutricionista · chef · jefe

Tarifas

Precio por dieta (default configurable). Solo admin edita.

admin · lectura otros

Alergias

Captura y consulta; resolución texto→catálogo.

auxiliar · enfermera · doctor

5Motores clave (la lógica que importa)

  • Motor de alergias (server/alergias.ts): una alergia a un alimento se expande a todo su grupo Codex — alérgico a la leche ⇒ evita todos los lácteos. La cocina marca las líneas en conflicto. La captura resuelve texto libre → alimento del catálogo para que igual dispare el bloqueo.
  • Máquina de estados (server/estados.ts): valida cada transición; solo permite el camino definido. Con bitácora en evento_dieta.
  • Regla D3 anti-duplicado (routers/enfermeria.ts): re-indicar el mismo paciente+tiempo reemplaza la bandeja si aún no está en cocina; si ya está en preparación o más, la bloquea (evita doble bandeja).
  • Aislamiento por sede (server/tenancy.ts): filtra por sede salvo roles transversales.

8Cómo analizarlo / correrlo

git clone https://github.com/eaguirre94/avante-dietas cd avante-dietas pnpm install pnpm db:generate # genera el cliente Prisma pnpm --filter @avante/web dev # http://localhost:3000 → /login # verificación pnpm typecheck · pnpm test · pnpm --filter @avante/web build

Requiere apps/web/.env con DATABASE_URL (Postgres) y AUTH_SECRET. Las credenciales reales no están en el repo (van por Secrets).
